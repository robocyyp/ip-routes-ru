# ip-routes-ru

Набор статических маршрутов для роутеров **Keenetic / Netcraze**, которые позволяют отправлять трафик к сайтам/сервисам, работающим **только из РФ**, **в обход VPN** (через обычного интернет‑провайдера).

## Когда это нужно

Частый сценарий:

- На роутере настроен VPN, и **весь трафик** домашней сети идёт через него (ноутбуки, телефоны, ТВ, приставки и т. д.).
- Зарубежные сайты работают нормально, но **часть российских сервисов** перестаёт открываться/работать через VPN.
- Не хочется постоянно переключать устройства между “VPN” и “без VPN”.

Решение: добавить статические маршруты из `routes.bat` на интерфейс вашего **основного подключения к провайдеру**, чтобы трафик к нужным РФ‑ресурсам шёл напрямую, а всё остальное — через VPN.

## Что уже включено

Сейчас в списке есть маршруты для:
- популярных стриминговых площадок,
- гос. сайтов,
- бюджетных организаций,
- некоторых банков и сервисов.

Этого хватает для моих повседневных задач. Если у вас что-то не работает — создавайте **Issues**, будем дополнять список.

## Как добавить маршруты (Keenetic / Netcraze)

> Важно: это имеет смысл только если VPN **уже настроен** на роутере и **работает**.

<img width="1133" height="1269" alt="image" src="https://github.com/user-attachments/assets/a9df2ac6-9cb0-45c7-bed5-4edf32f885e6" />

1. Откройте веб‑интерфейс роутера и перейдите в **Static routes (IPv4)**:  
   `https://192.168.1.1/staticRoutes/ipv4`  
   (Адрес может отличаться — используйте ваш адрес панели управления.)

2. Нажмите **Импорт**.

3. В интерфейсе импорта выберите подключение/интерфейс, через который ваш роутер выходит в интернет **к провайдеру** (то есть “обычный интернет”, в обход VPN).

4. Выберите файл **`routes.bat`** и нажмите **Загрузить**.
Иногда после загрузки может появиться сообщение, что маршруты не загружены, значит может быть проблема с правами на файл **`routes.bat`**, попробуйте создать вручную файл у себя локально и вставить туда содержимое из репозитория, затем повторите загрузку.

5. Готово. Теперь:
   - “зарубежный” трафик продолжит идти через VPN,
   - а сервисы, которым нужен доступ **из РФ**, будут работать через обычного провайдера на всех устройствах в вашей сети.
